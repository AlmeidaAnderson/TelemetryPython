## Immediate Next Steps

### 1. Installation (5 minutes)

**Windows Setup:**
```bash
# Double-click or run:
install.bat

# This will:
# - Check Python version (3.8+ required)
# - Install all dependencies
# - Verify installation
```

**Manual Installation:**
```bash
pip install -r requirements.txt
```

### 2. First Run (1 minute)

**Launch the application:**
```bash
# Double-click or run:
run_analyzer.bat

# Or directly:
python telemetry_analyzer.py
```

The main window should appear with 6 analysis tabs ready.

### 3. Try the Examples (5 minutes)

**Generate synthetic test data:**
```bash
python example_usage.py
```

This will:
- Create a 5-lap synthetic session
- Generate realistic telemetry data
- Save to `sessions/synthetic_YYYYMMDD_HHMMSS/`
- Demonstrate all major features

**Then in the analyzer:**
1. File ‚Üí Open Log
2. Navigate to the synthetic session folder
3. Open `telemetry.tlog`
4. Analysis ‚Üí Configure Track (auto-detect)
5. Analysis ‚Üí Segment Laps
6. Explore all 6 analysis tabs!

### 4. Import Your FuelTech Data (2 minutes)

**With the LOG_EXAMPLE.csv:**
```bash
# In the analyzer:
File ‚Üí Import FuelTech CSV...
# Select: LOG_EXAMPLE.csv
```

The data will be automatically converted and displayed.

### 5. Connect Live Telemetry

**If you have telemetry hardware:**
```bash
# In the analyzer:
Session ‚Üí Connect to Telemetry...
# Enter: COM3 (or your port)

Session ‚Üí Start Recording
# Drive your laps

Session ‚Üí Stop Recording
```

## Tips & Tricks

### Performance
- Reduce visible time window for smoother plots
- Use decimation for large files (automatic)
- Close unused analysis tabs

### Analysis
- Use histogram view to find optimal operating ranges
- GG diagram shows traction limits
- Track map reveals braking points

### Data Quality
- GPS tolerance of 50m works for most tracks
- Increase tolerance for long tracks (>5km)
- Check for GPS signal quality in data

## Troubleshooting

### "Module not found" Error
```bash
pip install -r requirements.txt
```

### "Access Denied" on COM Port
- Close other applications using the port
- Check Device Manager for correct port number

### Plots Not Updating
- Check that data contains expected channels
- Verify timestamp column exists
- Try different analysis tab

### Memory Issues
- Large files (>100MB) may be slow
- Close and reopen application
- Consider splitting large sessions
